# Job Search

使用 `web_search` 工具搜索招聘信息，整理后推送给用户。

## 搜索策略

### 搜索平台
- Boss直聘
- 猎聘
- 领英

### 关键词组合
执行 **3 次搜索**（每次 count=8）：

1. `"研发效能" OR "工程效率" 招聘 深圳 社招` — 研发效能方向
2. `"AI工程师" OR "AI平台" OR "大模型" 招聘 深圳 社招` — AI 方向
3. `"研发效能" OR "AI工程师" OR "平台工程" 远程 社招` — 远程岗位

**重要筛选条件：**
- **仅限HR发布**：过滤猎头、个体户发布的岗位
- **仅限社招**：过滤校招、实习岗位
- 在搜索词中加入 `HR` 或 `人事` 来辅助筛选

## 输出格式

将结果合并去重后，整理为 Markdown：

```
## 📋 今日岗位推荐（2026-03-07）

### 岗位1: [岗位名称]
- **公司**: xxx
- **来源**: Boss直聘/猎聘/领英
- **发布人**: HR姓名（如显示）
- **发布时间**: YYYY-MM-DD（如显示）
- **地点**: xxx
- **薪资**: xxx
- **岗位JD**: 
[完整JD内容]

---

### 岗位2: [岗位名称]
...
```

**规则：**
- 从搜索结果的标题和摘要中提取岗位名、公司、地点、薪资、发布时间、发布人
- 如果信息不完整，只填写能确认的字段
- 过滤掉培训机构、猎头广告、非岗位内容
- **不显示链接**，只标注来源平台名称
- 包含完整岗位JD（从搜索结果摘要中提取）
- 按公司知名度或薪资排序

## 去重

使用 `memory/job-search-history.md` 记录已推送的岗位（用"公司+岗位名"作为标识）：

1. **搜索前**：用 `read_file` 读取 `memory/job-search-history.md`（不存在则跳过）
2. **过滤**：跳过已出现在历史中的岗位（公司+岗位名相同）
3. **搜索后**：用 `edit_file` 将新推送的岗位追加到该文件

历史文件格式：
```
# Job Search History
<!-- 每行一个"公司-岗位名"，最新的在最后 -->
字节跳动-研发效能工程师
阿里巴巴-AI平台工程师
```

如果所有结果都是重复的，回复"今日暂无新岗位"。

## 无结果处理

如果搜索无结果或全部被过滤，简短告知用户即可，不要编造岗位信息。
